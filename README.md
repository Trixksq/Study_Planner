# Study Planner MVP

## Пітч

Багато студентів мають труднощі з організацією навчальних завдань і дедлайнів. 
Через це вони забувають виконувати домашні роботи або готуватися до контрольних.
Study Planner — це простий інструмент для планування навчальних завдань, який дозволяє 
додавати справи, переглядати список завдань та відмічати виконані. 
Це допоможе студентам краще організовувати навчання.

## Проблема

Студенти часто стикаються з такими проблемами:

- забувають про домашні завдання
- не бачать усі завдання в одному місці
- не планують навчання наперед

Через це навчальний процес стає менш ефективним.

## Ціль першої ітерації

Створити мінімальну версію програми, яка дозволить користувачеві:

- додавати навчальні завдання
- переглядати список завдань
- позначати завдання як виконані

## MVP

MVP включає основний сценарій використання:

1. Користувач вводить назву завдання.
2. Завдання додається до списку.
3. Користувач бачить список усіх завдань.
4. Користувач може позначити завдання як виконане.

Це мінімальний функціонал, який дозволяє перевірити ідею продукту.

## Карта стейкхолдерів

Основні зацікавлені сторони проєкту:

**Студенти**  
Основні користувачі програми, які використовують її для планування навчання.

**Розробник**  
Створює програму та реалізує її функціональність.

**Викладач**  
Оцінює результат виконання проєкту.

**Користувачі-тестувальники**  
Перевіряють зручність використання програми.

## Користувацький сценарій

**Початкові умови:**  
Користувач відкрив програму.

**Кроки:**

1. Користувач вводить назву навчального завдання.
2. Натискає кнопку "Додати".
3. Завдання з’являється у списку.
4. Після виконання користувач позначає завдання як виконане.

**Очікуваний результат:**  
Завдання додається до списку та може бути відмічене як виконане.

## Критерії приймання

Програма вважається працюючою, якщо:

- користувач може додати нове завдання
- завдання відображається у списку
- завдання можна позначити як виконане
- програма працює без помилок

## Definition of Success (Ознаки успіху)

Проєкт вважається успішним, якщо:
- користувач може виконати основний сценарій без допомоги
- програма стабільно працює під час тестування
- новий користувач може зрозуміти роботу програми за 1–2 хвилини
- основна функція (додавання завдання) працює без помилок

## План перевірки гіпотез

Для перевірки ідеї продукту:

- Дати програму кільком студентам.
- Попросити їх додати кілька навчальних завдань.
- Перевірити, чи можуть вони легко виконати основний сценарій.
- Отримати короткий відгук про зручність використання.

## Наступні кроки розвитку

У майбутньому можна додати:

- дедлайни для завдань
- нагадування
- сортування списку
- мобільну версію застосунку
